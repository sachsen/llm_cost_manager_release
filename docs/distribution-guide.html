Agent Meter

Codex・Claude Code／GLM・GitHub Copilotの利用状況を、このPC内で確認するWindows向け監視ツールです。

表示される割合は「使用済み」です。使用済み25%なら、残りは約75%です。

1. 導入

  1. GitHub ReleaseからZIPと同名の.sha256ファイルを取得し、SHA256が一致することを確認します。
  2. ZIPを任意の専用フォルダーへ展開します。
  3. agent-meter-desktop.exeとagent-meter-hook-helper.exeを同じフォルダーに置いたままにします。
  4. agent-meter-desktop.exeを起動します。インストーラーや管理者権限は不要です。
  5. Codexでは一度/hooksを開き、Agent MeterのSessionStartフックを確認して信頼します。Claude Code側は次回セッションから自動で有効です。

初回起動ではCodexとClaude Codeのグローバル設定へAgent MeterのSessionStartフックを既存項目とマージし、グローバルAGENTS.md / CLAUDE.mdへ専用マーカー付きの案内を追記します。以後はPC再起動後でも、エージェントの新規・再開セッション開始時にAgent Meterがタスクトレイへ自動起動し、エージェント自身も用途と参照コマンドを認識します。既存フックと案内以外の本文は保持されます。

現在のWindowsバイナリにはコード署名がありません。Windowsの警告が出た場合は、入手元とSHA256を確認できない限り実行しないでください。

2. 使い方

画面を閉じてもタスクトレイで監視を続けます。再表示はトレイアイコンの「Agent Meterを開く」、完全終了は「終了」です。表示は15秒ごと、Codexローカル統計は約60秒ごとに更新されます。「今すぐ取得」はCodexの契約枠を再取得します。

エージェントやPowerShellから現在値を確認するには、展開フォルダーで.\agent-meter-hook-helper.exe --statusを実行します。停止中ならバックグラウンド起動し、ダッシュボードと診断をJSONで返します。

期間別統計

10分・1時間・1日・7日・30日のプリセット、開始・終了日時、グラフ上の左右ハンドルで期間を選べます。「すべて」またはプロバイダー別に絞り込めます。

3. 対応範囲

対象取得方法制限
Codexapp-serverの契約枠と、ローカルsessionのtoken_countのみ初回は過去履歴も集計するため値が大きくなる場合があります。
Claude Codestatus lineとローカルセッションJSONLのusageメタデータ本文・ツール内容は保存せず、入力・出力・キャッシュトークンを60秒間隔で集計します。
GLM Coding PlanZ.AI公式glm-plan-usageと同じUsage API＋Claudeローカルusage5分間隔で契約枠と時間別・モデル別総トークンを取得し、ローカル履歴から内訳を取得します。
GitHub CopilotVS Code／Insiders／Cursor／Windsurf拡張の導入検知個人IDEのリアルタイム残量を取得する公式APIはありません。組織向け日次Usage Metricsとは別です。
「未報告」は障害とは限りません。提供元がその値を公開していない場合、Agent Meterは推測値を実測値として表示しません。

4. 保存するもの・しないもの

保存対象はトークン数、使用率、モデル名、リセット時刻、ハッシュ化した作業場所などです。プロンプト本文、回答本文、ツール引数・出力、APIキー、OAuthトークン、Cookieは保存しません。GLM認証値は実行時だけ読み、子プロセスの環境変数で公式APIへ渡します。DB・ログ・コマンドラインには保存しません。collectorは127.0.0.1:4318だけで待ち受けます。

データベースはWindowsのアプリデータ内に保存されます。ZIPや実行ファイルのフォルダーには実データを書きません。

5. トラブルシューティング

localhost接続拒否

Agent Meterを完全終了し、他のAgent Meterが残っていないことを確認して再起動します。ポート4318を別アプリが使用している場合も起動できません。

Claudeが更新されない

会話を一度実行し、最大約60秒待ってください。診断欄のclaude_local_usageがokならローカルusage読取は成功です。status lineはコンテキスト表示の補助経路です。

GLMが更新されない

Claude Code設定の接続先がZ.AIまたはBigModelで、GLM Coding Planの認証が設定されていることを確認してください。診断欄のglm_plan_usageがokなら公式総トークンと契約枠の取得成功です。

統計が0

対象期間とプロバイダー絞り込みを確認します。CodexとClaudeローカルusageは最大約60秒、GLM公式統計は最大約5分待ちます。Copilot個人利用は導入検知のみです。

「configured」と「ok」

configuredは設定を書き込んだだけで、受信成功を意味しません。okがデータ取得成功です。エラーコードがある場合は説明と一緒に配布者へ連絡してください。

6. 更新とバックアップ

トレイから終了し、新しいZIPを別フォルダーへ展開して起動します。統計DBはアプリデータ側にあるため通常は引き継がれます。Claude設定は初回連携時に%USERPROFILE%\.claude\settings.json.agent-meter-backup、既存のCodexフック設定は%USERPROFILE%\.codex\hooks.json.agent-meter-backupへ一度だけバックアップされます。既存のグローバルAGENTS.md / CLAUDE.mdも同じ場所へ.agent-meter-backup付きで一度だけバックアップします。新版を起動するとAgent Meterのフックコマンドと専用案内だけが新版のパスへ更新されます。

7. アンインストール

ZIP内のPowerShellを開き、用途に応じて実行します。通常実行でもCodex・Claude CodeからAgent Meterのフックと専用案内だけを除去し、他のフックと本文を保持します。

データを含めて削除

powershell -ExecutionPolicy Bypass -File .\uninstall-agent-meter.ps1

統計データを残す

powershell -ExecutionPolicy Bypass -File .\uninstall-agent-meter.ps1 -KeepData

Claude設定全体を導入前へ戻す

powershell -ExecutionPolicy Bypass -File .\uninstall-agent-meter.ps1 -RestoreClaudeSettings

通常実行ではAgent Meterのstatus line、フック、専用マーカー付き案内だけを削除し、連携前のstatus lineがあれば戻します。Claude設定全体の復元時は現在の設定をsettings.json.before-agent-meter-uninstall-日時へ退避してから、初回バックアップを復元します。最後に展開したAgent Meterフォルダーを削除してください。

-RestoreClaudeSettingsは設定全体を戻します。Claude Code設定をAgent Meter導入後に変更した場合、その変更が失われる可能性があるため、退避ファイルを確認して必要な項目を統合してください。

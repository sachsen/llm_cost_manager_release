Agent Meter for Claude Code

Claude Codeの利用状況だけをPC内で確認するWindows向け専用版です。

CodexとGLMの通信・収集・設定変更・表示はありません。Claude Code上でGLM互換モデルを使ったセッションも集計しません。

1. 導入

  1. Releaseからagent-meter-claude-only-windows-x64-<version>.zipと同名の.sha256を取得し、SHA256を照合します。
  2. フル版を実行中ならトレイから終了します。
  3. ZIPを専用フォルダーへ展開し、2つのEXEを同じ場所に置いたままagent-meter-desktop.exeを起動します。
  4. Claude Codeで新しいセッションを開始します。

初回起動はClaude Codeの既存status lineとSessionStartフックへAgent Meter項目をマージし、CLAUDE.mdへ専用案内を追加します。Codex設定には触れません。フル版と専用版は同じClaude連携設定を使うため、同時利用せずどちらか一方を選んでください。

現在のWindowsバイナリにはコード署名がありません。入手元とSHA256を確認できない場合は実行しないでください。

2. 使い方

画面は15秒ごと、ローカルトークン統計は最大約60秒で更新されます。ウィンドウを閉じてもトレイで監視を続けます。完全終了はトレイメニューの「終了」です。

JSONで現在値を取得するには展開フォルダーで次を実行します。

.\agent-meter-hook-helper.exe --status

停止中の場合はバックグラウンド起動してからClaude Codeだけのダッシュボードと診断を返します。

3. 取得範囲

対象動作
Claude Codestatus lineのモデル・コンテキストと、ローカルJSONLのusageメタデータを取得
CodexAPI通信、履歴読取、フック設定、表示を行わない
GLMZ.AI／BigModel通信、設定・認証値読取、履歴集計、表示を行わない
GLM互換Claudeセッションモデル名がGLMで始まる場合は破棄
GitHub Copilot検出・表示を行わない

4. プライバシー

保存するのはトークン数、モデル名、コンテキスト、取得状態などです。プロンプト本文、回答本文、ツール引数・出力、認証情報は保存しません。専用collectorは127.0.0.1:4319だけで待ち受け、データはWindowsアプリデータ内のclaude-only専用領域へ保存します。

5. トラブルシューティング

データが0のまま

Claude Codeで会話を一度実行し、最大約60秒待ちます。GLM互換モデルのセッションは仕様どおり表示されません。

localhost接続拒否

専用版を終了して再起動し、ポート4319を別アプリが使っていないか確認します。

フル版が起動する

Claude Code設定は最後に起動した版のhelperを参照します。両方を終了し、使用したい版のagent-meter-desktop.exeだけを起動してください。

6. アンインストール

変更対象を確認:

powershell -ExecutionPolicy Bypass -File .\uninstall-agent-meter.ps1 -WhatIf

データを含めて削除:

powershell -ExecutionPolicy Bypass -File .\uninstall-agent-meter.ps1

統計を残す場合は-KeepDataを付けます。専用アンインストーラーはClaude Code連携と専用データだけを対象とし、Codex設定には触れません。

# Agent Meter プライバシー情報

Agent Meterはローカルファーストの利用状況モニターです。取得した統計は利用者のPC内に保存されます。

## 保存する情報

- 入力、出力、キャッシュ、推論などのトークン数
- 提供元が報告した使用率とリセット時刻
- 使用モデル名
- 取得経路の状態と最終観測時刻
- 生の絶対パスではなく、ハッシュ化した作業場所と最終フォルダー名

## 保存しない情報

- プロンプト本文、回答本文
- ツールの引数や出力
- APIキー、OAuthトークン、Cookie
- Claude CodeやCodexの認証情報

## 読み取り範囲

- Codexのローカルセッションから、利用量に必要な`token_count`情報だけを読み取ります。
- Claude CodeのローカルセッションJSONLから、`usage`メタデータだけを読み取ります。
- Claude CodeがZ.AI／BigModel接続を使用している場合、GLM Usage APIの呼び出しに必要な認証値を実行時だけ参照します。

認証値はデータベース、ログ、コマンドラインへ保存しません。GLM APIを呼び出す子プロセスの環境変数としてだけ渡します。

## ローカル通信

collectorは`127.0.0.1:4318`だけで待ち受けます。内部取り込みAPIはAgent Meterが生成したBearerトークンで保護します。このトークンを公開したり、サポート依頼へ貼り付けたりしないでください。

## 保存場所

統計データと連携用の内部ファイルはWindowsのAgent Meterアプリデータ内に保存されます。配布ZIPや展開フォルダーには実データを保存しません。

## 設定変更とバックアップ

初回起動時、CodexとClaude Codeのグローバル`SessionStart`フックへAgent Meter専用項目をマージし、グローバル`AGENTS.md`／`CLAUDE.md`へ専用マーカー付き案内を追記します。

変更前の設定・案内ファイルは、同じ場所へ`.agent-meter-backup`付きで一度だけバックアップします。アンインストーラーはAgent Meter専用項目だけを除去する通常モードと、Claude Code設定全体を初回バックアップへ戻す明示的なモードを備えています。

## サポート時の注意

不具合報告へ認証ファイル、トークン、プロンプト、会話ログを添付しないでください。診断欄の状態名とエラーコードだけで再現できない場合は、秘密情報を除去した最小限の情報を使用してください。
